# OpenStack Network 구성

---

## Provider Network

- **Provider Network**는 OpenStack 외부의 **물리 네트워크와 직접 연결**되는 방식입니다.
- 인스턴스는 생성과 동시에 외부망(IP)을 직접 할당받으며, **별도의 라우터나 NAT 없이 외부와 통신**할 수 있습니다.
- **네트워크 생성은 오직 관리자(admin)가 가능**하며, 주로 flat 또는 VLAN 타입으로 구성됩니다.
- 보통 **단일 테넌트**, 고정 IP, 단순한 구성에 적합하며, **구성이 간단**하다는 장점이 있습니다.
- 그러나 **테넌트 간 네트워크 격리가 어렵고**, 대규모 멀티테넌트 환경에는 적합하지 않습니다.

---

## Self-Service Network

- **Self-Service Network**는 **프로젝트 사용자(tenant)**가 직접 네트워크, 서브넷, 라우터 등을 생성할 수 있는 구조입니다.
- 기본적으로 **VXLAN 또는 GRE 기반의 Overlay 네트워크**를 통해 격리된 가상 네트워크를 제공합니다.
- 외부 통신은 라우터를 통해 **NAT/Floating IP**를 사용하며 이루어집니다.
- DHCP, 보안 그룹, Floating IP, NAT, 라우터 등 다양한 기능을 제공해 **확장성과 유연성이 매우 높습니다**.
- **대규모 클라우드 환경**, 멀티 프로젝트, SaaS/Dev 환경 등에 적합하며, 관리자의 개입 없이도 다양한 네트워크 구성이 가능합니다.
- 단, 내부 네트워크만으로는 외부와 직접 통신할 수 없기 때문에, **Floating IP를 반드시 할당**해야 외부 접근이 가능합니다.

---

> 현재 네트워크 구성은 Self-Service Network로 구성함
